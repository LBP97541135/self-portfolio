## 5. 内容来源（用于后续改稿时参考）

下面这些信息**不直接展示在作品集前端**，只作为后续维护本条项目时的素材库，方便补充 / 改写上面那段 Mock 数据。

### 5.1 核心展示意图

这个项目在 2-3 分钟内要让访客理解：

1. 这是一个什么产品（AI 互动叙事 / 剧本杀）。
2. 用户如何选剧本、配 Agent、开始游戏。
3. Agent 在游戏中如何扮演玩家 / DM / 陪玩。
4. 调查、对话、出示证物、推进阶段的玩法。
5. 复盘阶段 Agent 如何从经验中进化。

### 5.2 产品六层结构

- 剧本库：6 个剧本（雪夜钟声、第七位继承人、红绒幕后的吻、消失的调酒师、午夜列车证词、旧宅来信）。
- Agent 阵容：8 个 Agent（洛白、祁灯、阿砚、林雾、旧钟、鸦青、南栀、Echo）。
- 游戏舞台：阶段 setup → intro → script_reading → investigation → deduction → voting → reveal → review。
- 证物推理：公开 / 已发现 / 隐藏三类证物，调查 → 出示 → 角色反应。
- 复盘进化：评分、关键时间线、Gene / Capsule 沉淀。
- 个人助手：用户画像 + 下一局推荐。

### 5.3 Agent 分类

陪玩 / DM / 推理 / 情绪 / 破局 五类，其中：

- 旧钟是 DM 节奏控制型。
- Echo 专门负责复盘总结。
- 南栀负责新手陪伴。
- 鸦青是反常识破局型。

### 5.4 进化机制

- 每局结束后，Agent 改写 constitution 摘要。
- 新增 memory capsule，记录可复用策略。
- 下次开局前由个人助手推荐最合适的剧本 + 阵容。
