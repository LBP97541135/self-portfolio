01 / Profile

只做有工程支撑
的产品

我拒绝停留在 "AI Demo" 的表面。在不确定的业务场景中，我通过拆解复杂流程、设计确定性机制，并利用前端工程将这些能力转化为直观、可信的用户体验。

从商家工单自动化到行业级路线规划，我始终坚持：最好的 AI 产品，是让技术在解决问题的过程中变得“隐形”。

  • Agent 工程 01

    从自研 Multi-Agent 框架（3 种执行模式、Skill 向量检索、Memory 持久化）到企业级落地（Darwin / AgentScope / Langflow），熟悉 React 自主模式、Human-in-the-Loop 设计与 Fleet 批量评测。

  • 全栈交付 02

    Java / Python / TypeScript 均有独立交付经验，能从数据库设计、后端 API 到前端交互独立完成闭环，不依赖拆分协作。

  • 产品设计 03

    习惯从用户问题出发定义需求，而非从技术出发找场景。有从 0 推动项目落地的经验，包括工单自动化的业务主推和 Work Coach 的完整产品逻辑设计。

  • 评测与优化 04

    相信可量化才能持续优化。建立过双层视觉评测体系（11% → 1.7%）、Fleet 批量博弈评测、任务拆分使小模型达到 98% 准确率，习惯用数据说话。

  • 协议与集成 05

    手写实现 MCP Server（stdio / SSE / Streamable 三协议），集成过税务局 API、企微多维表、钉钉 DEAP、Darwin 框架等多种外部系统，私有化部署有实战经验。

  • 跨域落地 06

    在制造、金融、能源、航运、科研、娱乐等不同行业有交付经验，能快速理解业务逻辑并将 AI 能力翻译成可用方案，不局限于单一场景。

04 / Thesis

关于 AI 产品的
核心主张

01 / 精准定位 AI 边界

不是所有流程都需要智能体。我优先评估业务约束与失败成本，确保 AI 仅出现在能产生确定性收益的环节。

02 / 拆解模糊为执行单元

我将感性的需求转化为理性的角色、任务与工具链。通过设计反馈机制，让系统在自主行动与人工干预间达成平衡。

03 / 以工程交付可验证的价值

交付不仅仅是演示。我通过核心指标（如准确率、提效比、任务完成率）来衡量每一个 AI 功能的实际业务贡献。
